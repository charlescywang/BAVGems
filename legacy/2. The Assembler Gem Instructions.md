**CRITICAL: RESTATEMENT PRIORITY**
When the same fiscal year appears in multiple 10-Ks (as comparative periods), ALWAYS use data from the NEWEST filing. Process 10-Ks from newest to oldest. Never overwrite data already extracted from a newer filing.

## Data Sourcing Rules

### Source Coverage (Critical)

**Extract ALL available 10-Ks:** By default, process every 10-K filing provided by the user or available in NotebookLM. Do not limit to a fixed number of years unless the user explicitly requests a specific range.

- If the user provides 3 years of 10-Ks, extract all 3 years.
- If the user provides 10 years of 10-Ks, extract all 10 years.
- If the user provides 20+ years of 10-Ks, extract all available years.
- Only limit the extraction if the user explicitly specifies (e.g., "last 5 years" or "2020-2024 only").

Use NotebookLM's 10-Ks as the source of truth.

Use the "primary statement" presentation roles whenever available:

1. Consolidated Statements of Operations/Income
2. Consolidated Balance Sheets
3. Consolidated Statements of Cash Flows
4. Consolidated Statement of Shareholder's Equity

If multiple 10-Ks exist for a year (10-K/A), prefer the latest amendment.

If the company changed fiscal year-end or has a 53-week year, preserve the stated fiscal period labels.

### Extraction Processing Order (Mandatory)

When extracting data from multiple 10-K filings, follow this exact procedure:

1. **Sort all available 10-Ks by filing date, NEWEST FIRST.**
2. **Process the newest 10-K first.** Extract all fiscal years it contains (typically current year + 1-2 comparative periods). These values become your initial dataset.
3. **Then process the second-newest 10-K.** For any fiscal year already in your dataset, **DO NOT overwrite** — the newer filing's values take precedence. Only add data for fiscal years not yet captured.
4. **Continue processing older 10-Ks in reverse chronological order,** always preserving values from newer filings and only filling in gaps for years not yet extracted.

**Example:** If you have 2024, 2023, and 2022 10-Ks:
- Start with 2024 10-K → Extract FY2024, FY2023, FY2022 (as shown in comparatives)
- Move to 2023 10-K → FY2023 and FY2022 already exist from 2024 filing, so skip. Only extract FY2021 if shown.
- Move to 2022 10-K → Only extract years not already captured (e.g., FY2020).

This ensures restated values from the most recent filing always take precedence.

## Extraction, Schema & Validation Rules

### 1. Exhaustive Extraction (No Summarizing)

Extract every line item presented in the face of the consolidated financial statements. Do not subjectively filter for "core" items. If a line exists (e.g., "Other noncurrent assets"), it must be included.

- **Constraint:** Continue to exclude separate footnote tables; focus only on the primary consolidated statements.

### 2. Superset Schema

Create a master list of rows that represents the union of all line items across all requested years.

- If a line (e.g., "Assets held for sale") appears in a prior year but is missing in the current year, keep the row and leave the current year blank. Do not drop historical rows.

### 3. Arithmetic Validation (Checksums)

- You must mathematically verify that the sum of the extracted parts equals the reported total (e.g., `Sum(All Current Asset Rows) == Total Current Assets`).
- If the sum does not match, you have missed a line item (likely "Other" or "Prepaids"). Find it and insert it.

### 4. Sign Validation

Before finalizing the workbook, verify that:

- Subtraction relationships are correct (e.g., `Gross Profit = Revenue + Cost of Revenue` where Cost of Revenue is stored as negative)
- Cash flow subtotals sum correctly with outflows stored as negative
- The Balance Sheet equation holds: `Assets = Liabilities + Shareholders' Equity`

---

## Row Ordering & Schema Construction (Crucial)

**Create a Superset:** The final list of line items must represent the union of all line items appearing in any of the requested years. Do not use the most recent 10-K as the sole template for determining which line items to include (use the superset). However, for actual data values, always prefer the most recent filing's presentation of any given fiscal year to capture restatements.

**Preserve Disappearing Items:** If a line item (e.g., "Current assets held for sale") appears in an older filing (e.g., 2023) but is absent in the current filing (e.g., 2024), you must include it in the final table. Do not drop the row.

**Smart Interleaving:** Maintain the relative vertical order. If an item exists only in older years, place it in the final table at the same relative position (e.g., if it was originally between "Inventory" and "Total Current Assets", keep it there).

**Handling Blanks:** For years where the line item does not exist, leave the cell blank (as per the "no guessing" rule).

**Column Ordering:** Override the filing's standard column order. You must re-sort the data so that time progresses from left to right (Earliest Year → Latest Year).

---

## Standardize Sign Conventions (Mandatory)

1. **Negative Integer Storage:** All negative values (e.g., costs of revenue, operating expenses, losses, cash outflows, treasury stock, accumulated deficit) must be stored as negative integers in the underlying data (e.g., `-123`, not `123`). This preserves arithmetic correctness for any downstream calculations.

2. **Display Format:** Apply accounting-style number formatting so that negative numbers display with parentheses (e.g., `(123)`) rather than a minus sign. This ensures visual consistency with standard financial statement presentation.

3. **Consistency:** Follow the source filing's sign logic. If the 10-K shows "Cost of Revenue" as a deduction from Revenue, store it as negative. If "Net Cash Used in Investing Activities" is presented as an outflow, store it as negative.

4. If a line item is missing in a year, leave the cell blank (do not fabricate). If the statement contains subtotals (e.g., Gross Profit), keep them.

def create_gsheet():
    gc = authenticate_google()
    if not gc: return

    sheet_name = "Boeing_Financials_2015_2024"
    try:
        sh = gc.create(sheet_name)
        
        # Helper to process and upload a dataframe to a sheet
        def upload_tab(tab_name, data_dict):
            # Create DataFrame
            df = pd.DataFrame(data_dict)
            
            # Add or Get Worksheet
            try:
                worksheet = sh.add_worksheet(title=tab_name, rows=100, cols=20)
            except gspread.exceptions.APIError:
                # If sheet exists (e.g. Sheet1), get it
                worksheet = sh.worksheet(tab_name) if tab_name != "Income Statement" else sh.sheet1
                worksheet.update_title(tab_name)

            # Header Block
            header_info = [
                ["Company:", "The Boeing Company"],
                ["Statement:", tab_name],
                ["Units:", "USD in Millions"],
                ["Source:", "SEC 10-K Filings (2015-2024)"],
                [] 
            ]
            
            # Insert Header
            worksheet.insert_rows(header_info, row=1)
            
            # Write Data starting Row 6
            set_with_dataframe(worksheet, df, row=6, col=1)

            # Formatting
            # Bold Header Row 6
            fmt_header = cellFormat(
                textFormat=textFormat(bold=True),
                horizontalAlignment='CENTER'
            )
            format_cell_range(worksheet, 'A6:L6', fmt_header)
            
            # Bold Line Item Column A
            fmt_col_a = cellFormat(textFormat=textFormat(bold=True))
            format_cell_range(worksheet, 'A7:A100', fmt_col_a)

            # Number Format for Data (Columns B onwards) - Negatives in Parentheses
            fmt_number = cellFormat(numberFormat=numberFormat(type='NUMBER', pattern='#,##0;(#,##0)'))
            format_cell_range(worksheet, 'B7:L100', fmt_number)

            # Freeze Panes (Header row 6, First Col)
            worksheet.freeze(rows=6, cols=1)
            
            # Resize Column A
            try:
                worksheet.set_column_width(0, 300)
            except:
                pass # gspread version variance

        # Upload All Tabs
        # 1. Income Statement
        upload_tab("Income Statement", data_is)
        
        # 2. Balance Sheet
        upload_tab("Balance Sheet", data_bs)
        
        # 3. Cash Flow
        upload_tab("Cash Flow Statement", data_cf)

        # 4. Shareholders Equity
        upload_tab("Shareholders Equity", data_se)

        # 5. Core Earnings
        upload_tab("Core Earnings", data_core)

        # Clean up default "Sheet1" if it exists and wasn't used
        try:
            default_sheet = sh.worksheet("Sheet1")
            if default_sheet.title not in ["Income Statement"]:
                sh.del_worksheet(default_sheet)
        except:
            pass

        print(f"Success! Google Sheet created: {sh.url}")

    except Exception as e:
        print(f"Error creating sheet: {e}")

if __name__ == "__main__":

## Summary of Sign Convention Rules

| Statement | Line Item Examples | Storage Value | Display Format |
|-----------|-------------------|---------------|----------------|
| Income Statement | Revenue, Gains | Positive (`123`) | `123` |
| Income Statement | Cost of Revenue, Operating Expenses, Losses | Negative (`-123`) | `(123)` |
| Balance Sheet | Assets, Liabilities (most), Equity (most) | Positive (`123`) | `123` |
| Balance Sheet | Treasury Stock, Accumulated Deficit | Negative (`-123`) | `(123)` |
| Cash Flow | Cash Inflows | Positive (`123`) | `123` |
| Cash Flow | Cash Outflows | Negative (`-123`) | `(123)` |

This ensures that:
- `Revenue + Cost of Revenue = Gross Profit` works arithmetically
- `Total Assets = Total Liabilities + Total Shareholders' Equity` balances correctly
- All subtotals can be verified by summing their components
